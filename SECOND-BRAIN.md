# 第二大腦系統手冊

> 一句話定位：**現場採集原始素材 → AI 轉文字/OCR → 沉澱進策略地圖 Wiki →
> 用 claude.ai 直接問答**。四塊各自獨立運作，互不依賴，Wiki 是唯一的
> 匯流點。這份文件是給人看的操作手冊；更偏工程取捨的技術筆記在
> [`ARCHITECTURE.md`](ARCHITECTURE.md)。

## 一、整體長相

```
  現場（展會／拜訪／實驗／上課）
         │
         ├─ 隨身記 fieldlog ─────┐
         │  （個人，行動優先）      │
         │                       ├──▶ 策略地圖 Wiki ──▶ claude.ai
         └─ Medtec 參展系統 ──────┘   （人審 git diff）    （MCP 問答窗口）
            （8人團隊共筆）              ▲
                                        │
                              LitDB（文獻／專利，尚未建置）
```

- **隨身記**與**參展系統**是「採集層」：各自獨立的 Cloudflare Worker、
  各自的資料庫，互不影響，也刻意不合併
- **策略地圖 Wiki**是「知識層」：純 Markdown、git 版控，AI 只能提案、
  人看 diff 才收錄
- **MCP（`medapi-mcp`）** 是「問答層」：唯讀窗口，讓 claude.ai／Claude
  Code 直接用自然語言跨三個來源查資料，不動任何原始系統

## 二、四個組成部分

### 1. 隨身記（fieldlog）—— 個人現場採集

**做什麼**：出差／拜訪／實驗／上課時，錄影、拍照、錄音、速記。每張照片
自動標「錄音第幾分幾秒拍的」，錄音每 10 分鐘自動分段即傳，斷網也能先存
手機再補傳。

**網址**：`https://fieldlog.gogoyankee.workers.dev`（PIN 登入，可加到手機
主畫面變成 App）

**資料流**：現場採集 → 進收件匣 → 有空時「🪄 一鍵整理」批次轉文字＋OCR
→ 歸檔到資料夾 → 資料夾按「匯出給 AI」產出一份 Markdown 原料包 → 貼給
Claude/GPT 說「彙整成報告」→ 存進 Notion（目前這一步是人工貼上，自動
同步的欄位已經在但沒接通，見下方「待辦」）

**Wiki 閱讀入口也在這裡**：首頁「🗺 知識庫」連到 `/wiki.html`，跟隨身記
共用同一組 PIN。

### 2. Medtec 參展系統（cloudflare/）—— 團隊共筆

**做什麼**：8 位同事出發前瀏覽 585 家展商名單、篩選分類、留言洽談；
現場記錄拜訪狀態、部門標籤、資質確認、拍照錄音。

**網址**：`https://medtec-2026.gogoyankee.workers.dev`

**跟隨身記的差別**：這是多人即時協作的共筆工具（誰指派給誰、誰拜訪過、
LINE 每日摘要推播），隨身記是單人行動採集，兩者資料結構天生不一樣，
所以刻意不合併資料庫。

**團隊功能**：分派清單、口袋名單、CSV 匯出、每日 LINE 摘要（選用）、
展商詳情頁一樣有「🪄 一鍵整理」批次轉文字/OCR。

### 3. 策略地圖 Wiki —— 知識沉澱層

**做什麼**：長儒的個人知識庫，把上面兩個採集系統的原始素材，編譯成
可疊代的技術論述。純 Markdown 檔（`fieldlog/public/wiki/`），git 記錄
每一次修改。

**閱讀入口**：隨身記首頁的「🗺 知識庫」（`/wiki.html`）。

**條目地圖**：
| 分類 | 條目 |
|---|---|
| A. 核心技術主題 | A1 親水披膜／A2 抗結痂披膜／A3 抗菌披膜（各自對應產品） |
| B. 支撐知識庫 | B1 塗層材料化學／B2 表面處理製程／B3 性能測試失效分析／B4 生物安全與法規（跨產品共用內容） |
| C. 資源網絡 | C1 供應商與設備圖譜（進料＝參展系統）／C2 文獻與課程（進料＝隨身記＋文獻閱讀） |

**收錄鐵律**：跨產品都成立的內容只寫一份在 B 頁，A 頁牽涉到時用一兩句
話帶過＋連結指過去，不整段複製。AI 只能改寫提案、commit，**人看 git
diff 決定收錄／修改／退回**才算數。

### 4. MCP 問答窗口（medapi-mcp）—— claude.ai 直接問

**做什麼**：讓 claude.ai（或 Claude Code）當你的窗口，一次問跨三個
來源的問題，例如「展商裡做親水塗層的有哪幾家」「上次拜訪百賽飛聊到
什麼」「wiki 的抗結痂條目寫到哪」。

**網址**：`https://medapi-mcp.gogoyankee.workers.dev`（本身不是給人瀏覽
的頁面，是給 claude.ai 連的 MCP 端點）

**鐵律：全唯讀**。只查資料，不寫入、不刪除——要改資料還是回各自的
前台操作，wiki 收錄還是走 git 人審。技術上用 Cloudflare 的 Service
Binding 直接呼叫隨身記／參展系統的 Worker（不透過對外網路，也因此不受
兩邊前台怎麼改版影響），D1 則是唯讀共綁。

## 三、操作步驟

### A. 現場採集（展會／拜訪／實驗／上課）
1. 打開隨身記或參展系統（依場景挑一個），按對應的按鈕開始錄影／拍照／
   錄音／記事
2. 現場全程可以只顧著看展/做實驗，不用中斷去打字——先採集，事後整理
3. 結束後資料會先進收件匣（隨身記）或該展商的附件區（參展系統）

### B. 一鍵整理（轉文字＋OCR）
1. 有空時打開該筆紀錄/展商，按「🪄 一鍵整理」
2. 按鈕旁會顯示「⏳ N 筆待整理」——**如果顯示「✓ 已全部整理」按鈕會反灰**，
   代表沒有新東西要處理，不用按
3. 系統只處理「還沒轉文字的錄音」跟「還沒擷取文字的照片」，已經處理過的
   不會重跑，不浪費 Workers AI 額度
4. 兩個系統的參展系統首頁會顯示「🪄 今日 AI 已處理 X 筆」，是兩邊
   （隨身記＋參展系統）共用同一份 Cloudflare 免費額度（約 10,000
   Neurons/天）的粗略用量參考，額度快用完時會有感

### C. 更新 Wiki
1. 事件驅動：展會結束／課程結束／實驗告一段落後，跟 AI（我）說一聲
   「更新 wiki」
2. AI 會抓「上次更新後」的新採集資料（隨身記＋參展系統裡已經轉好文字/
   OCR 的內容），改寫對應條目、commit
3. 看 git diff：覺得對就說收錄，覺得哪裡不對就說怎麼改或退回

### D. 用 claude.ai 直接問
1. claude.ai → Settings → Connectors，確認 `MyWiki`（或你取的名字）
   連接器狀態是連線中
2. 直接在對話裡問問題即可，不用其他設定；第一次呼叫某個查詢工具時
   會跳出「允許使用」確認，之後照連接器設定的權限模式走
3. 想在 Claude Code 裡用也可以，同一個帳號下的連接器這邊也看得到

### E. 部署與維運（改程式碼之後）
1. 三個 Worker（`fieldlog`、`medtec-2026`、`medapi-mcp`）都是接 GitHub
   的自動部署——**push 到這個分支就會自動重新部署**，不用手動觸發
2. 去 Cloudflare Dashboard 對應 Worker 的 **Deployments** 頁可以看部署
   log／狀態
3. 三個 Worker 各自的 Secret／Variable 在該 Worker 的 **Settings →
   Variables and Secrets** 管理：
   | Worker | 需要的 Secret/Variable |
   |---|---|
   | `fieldlog` | `FIELD_PIN` |
   | `medtec-2026` | `TEAM_PIN`（另有選用的 LINE 相關 Secret） |
   | `medapi-mcp` | `MCP_PIN`、`FIELD_PIN`（跟 fieldlog 同值，讀 wiki 用） |
4. 想知道今天 AI 額度用得如何：直接看參展系統首頁的用量提示，不用去
   Cloudflare 後台查
