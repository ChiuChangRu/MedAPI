# 策略地圖 Wiki（長儒的個人知識層）

> 這不是文件堆，是**活的知識庫**：Medtec 參展系統與隨身記負責採集原始素材，
> AI 定期把新素材編譯進對應條目，人審核 git diff 後收錄。
> 原則：原始證據不可變（存在 R2）、wiki 可疊代（git 記錄每次版本）、
> 每個重要論述盡量標來源。

**閱讀入口**：隨身記前台的「🗺 知識庫」（`/wiki.html`，與隨身記共用 PIN）。
編輯一律走 git（本資料夾 `fieldlog/public/wiki/`），push 後自動部署。
新增條目時記得同步登記到 `pages.json`，前台選單才會出現。

## 更新流程（驗證關卡）

1. 觸發：事件驅動（展會日/課程/實驗結束後），跟 AI 說「更新 wiki」
2. AI 拉出「上次更新之後」的新採集資料，改寫對應條目、commit
3. 人看 git diff：收錄／修改／退回
4. 每個條目底部的「更新日誌」記錄：日期、資料來源、改了什麼

## 條目格式約定

每頁固定骨架：`主題定位 → 技術重點 → 素材與證據 → 開放問題 → 更新日誌`。
供應商以段落＋連結活在條目裡，不單獨開頁（防止條目爆炸）；
某家升格為重點合作對象時再開子頁。
